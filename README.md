# spaghetti-slicer

A frontend best practices auditor for React/TypeScript codebases — especially those generated by AI tools like Cursor, v0, Bolt, and Lovable.

## What is spaghetti-slicer?

spaghetti-slicer scans your React/TypeScript codebase and reports violations across 12 rules in 5 categories, with a score per category and a total score out of 100. It catches common anti-patterns before they reach production.

## Installation

```bash
# Run without installing
npx spaghetti-slicer ./src

# Install globally
npm install -g spaghetti-slicer
spaghetti-slicer ./src
```

## Local Development & Testing

If you are developing `spaghetti-slicer` locally and want to test it against another local project on your machine, you can run it using the following approaches:

### Option 1: Run via path directly
Build the tool, then execute the compiled JS bundle pointing to your other project's path:
```bash
# Build the project
npm run build

# Run the compiled CLI against another folder
node /path/to/spaghetti-slicer/dist/index.js /path/to/other-project/src
```

Or run directly from source using `tsx`:
```bash
npx tsx /path/to/spaghetti-slicer/src/cli/index.ts /path/to/other-project/src
```

### Option 2: Link globally via npm link
You can link the package globally to test it as a regular global command:
1. In the `spaghetti-slicer` directory:
   ```bash
   npm link
   ```
2. Now you can run the linked `spaghetti-slicer` command from any directory on your system:
   ```bash
   spaghetti-slicer /path/to/other-project/src
   ```
3. To unlink when you are done, run the following in the `spaghetti-slicer` directory:
   ```bash
   npm unlink -g
   ```

## Usage

```bash
# Audit a directory
spaghetti-slicer ./src

# Audit a single file
spaghetti-slicer ./src/components/App.tsx

# Output JSON (for CI pipelines or tooling)
spaghetti-slicer ./src --json

# Only run one category of rules
spaghetti-slicer ./src --rule=a11y
spaghetti-slicer ./src --rule=typescript

# Fail CI if score is below threshold
spaghetti-slicer ./src --min-score=80

# Show auto-fix suggestions
spaghetti-slicer ./src --fix
```

## Rules

### Architecture (weight: 25%)

| Rule | Severity | Description |
|------|----------|-------------|
| `component-length` | critical | Components exceeding 200 lines should be broken up |
| `business-logic-in-jsx` | warning | `.filter()`, `.reduce()`, `.sort()` inline in JSX |
| `direct-fetch-in-component` | critical | `fetch()`/`axios` called directly in a component body |

### React (weight: 20%)

| Rule | Severity | Description |
|------|----------|-------------|
| `index-as-key` | critical | Array index used as React `key` prop |
| `useeffect-missing-deps` | critical | `useEffect` called without a dependency array |
| `missing-error-boundary` | warning | No error boundary component found in codebase |

### TypeScript (weight: 20%)

| Rule | Severity | Description |
|------|----------|-------------|
| `no-explicit-any` | warning | Explicit `any` type annotation |
| `missing-return-type` | info | Exported functions without return type annotations |

### Accessibility / a11y (weight: 20%)

| Rule | Severity | Description |
|------|----------|-------------|
| `img-missing-alt` | critical | `<img>` elements missing `alt` attribute |
| `button-missing-aria-label` | critical | Icon-only buttons without `aria-label` |

### Performance (weight: 15%)

| Rule | Severity | Description |
|------|----------|-------------|
| `full-library-import` | warning | Default import of large libraries (lodash, moment, etc.) |
| `image-missing-dimensions` | warning | `<img>` or `<Image>` missing `width`/`height` (causes CLS) |

## Scoring

Each category starts at 100 and deductions are applied per violation:
- **Critical**: −10 points
- **Warning**: −5 points  
- **Info**: −2 points
- Minimum score per category: 0

**Total score** = weighted average across categories:

| Score | Grade |
|-------|-------|
| 90–100 | ✅ Excellent |
| 75–89 | ✅ Good |
| 60–74 | ⚠️ Needs Work |
| < 60 | ❌ Poor |

## CI Integration

Add to `.github/workflows/audit.yml`:

```yaml
name: Frontend Audit
on: [pull_request]

jobs:
  audit:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - name: Run spaghetti-slicer
        run: npx spaghetti-slicer ./src --min-score=75
```

The `--min-score` flag causes the process to exit with code 1 if the score falls below the threshold, failing the CI check.

## Contributing

1. Fork the repository
2. Add your rule in `src/rules/<category>/<rule-name>.ts` implementing the `Rule` interface
3. Register it in `src/rules/index.ts`
4. Add tests in `tests/rules/<category>/<rule-name>.test.ts` with at least a passing and failing fixture
5. Run `npm run test:run` to verify
6. Submit a pull request
